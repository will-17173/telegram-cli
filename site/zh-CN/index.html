一套命令界面

从在线聊天，到本地记忆。

明确选择何时连接 Telegram、何时只查询 SQLite，以及何时允许远端写入。每一种工作流的边界都清晰可见。

  1. 01
    ↯

    在线读取

    查看未读聊天、读取近期消息或监听消息流，查询结果无需写入本地数据库。

    tg inbox --markdown
  2. 02
    ⌁

    建立记忆

    把需要的聊天同步到 SQLite，之后无需重连即可搜索与分析。

    tg sync @team
  3. 03
    →

    谨慎操作

    用明确命令发送消息和管理群组，并通过全局开关控制所有远端写入。

    tg config write-access off

从这里开始

三个命令，看到第一份聊天列表。

Telegram CLI 需要 Node.js 22 或更高版本。账号认证会在终端内交互进行，每个账号的会话都保存在本地。

  1. 1 全局安装
    npm install -g @will-17173/telegram-cli
  2. 2 添加账号
    tg account add
  3. 3 列出聊天
    tg status
tg chats
接着选择一个聊天，建立本地历史 tg sync <chat> && tg search "keyword" --chat <chat>

清晰可见的安全边界

放心读取。
远端写入设一道闸。

全局关闭 Telegram 远端修改，同时保留在线读取和本地数据库命令。确实需要操作时，再有意识地重新打开。

tg config write-access off ↔ tg config write-access on

Telegram 权限和速率限制仍然适用。大量历史或媒体请求可能触发 flood wait。
